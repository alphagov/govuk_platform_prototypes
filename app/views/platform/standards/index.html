The Digital Service Standard

Government services are assessed against these 18 criteria. Find out more

The criteria:


  1. Start with user needs

    Use research to develop a deep knowledge of who the service users are and what that means for the design of the service.

  2. Create a research plan

    Plan for ongoing user research and usability testing so you can continuously improve the service.

  3. Create a multidisciplinary team

    You have a team that can design, build and operate the service, led by a skilled, senior service manager with decision-making responsibility.

  4. Be agile and user-centred

    Develop the service using an agile, iterative and user-centred methodology.

  5. Make continuous improvements

    Make sure you have the capacity, resources and technical flexibility to iterate and improve your service on a regular basis.

  6. Evaluate the technology choices

    Evaluated the tools and systems that will be used to build, host, operate and measure the service, and how to procure them.

  7. Evaluate security and privacy risks

    Evaluate the user data that the service provides or stores and address any associated security, legal and privacy issues and risks. Consult with experts where appropriate.

  8. Make things open

    Make all new source code open and reusable, and publish it under appropriate licences. Provide a convincing explanation in cases where this can't be done.

  9. Use open standards and platforms

    Use open standards and common government platforms where available.

  10. Test end-to-end

    Test the service end-to-end in an environment identical to the live version, including on all common browsers and devices, and using dummy accounts and a representative sample of users.

  11. Plan for downtime

    Make a plan for the event of the digital service being taken temporarily offline.

  12. Make it usable

    Create a service that is simple and intuitive enough that users succeed first time.

  13. Make it consistent

    Build a service consistent with the user experience of the rest of GOV.UK including using the design patterns and style guide.

  14. Encourage channel shift

    Encourage all users to use the digital service (with assisted digital support if required), alongside an appropriate plan to phase out non-digital channels/services.

  15. Measure performance

    Use tools for analysis that collect performance data. Use this data to analyse the success of the service and to translate this into features and tasks for the next phase of development.

  16. Establish KPI benchmarks

    Identify performance indicators for the service, including the 4 mandatory key performance indicators (KPIs) defined in the manual. Establish a benchmark for each metric and make a plan to enable improvements.

  17. Publish performance data

    Report performance data on the Performance Platform.

  18. Test with the minister

    Test the service from beginning to end with the minister responsible for it.
